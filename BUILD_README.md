# CxKitty 打包说明

本文档说明如何将 CxKitty 项目打包成 Windows 可执行文件 (.exe)。

## 环境要求

- Python 3.11 (推荐使用 3.11.x 版本)
- Windows 操作系统
- 至少 2GB 可用内存
- 至少 500MB 可用磁盘空间

## 打包方法

### 方法一：使用批处理文件（推荐）

1. 双击运行 `build.bat` 文件
2. 等待打包完成
3. 打包成功后，exe 文件将位于 `dist/CxKitty.exe`

### 方法二：手动执行

1. 打开命令提示符或 PowerShell
2. 切换到项目根目录
3. 执行以下命令：

```bash
# 安装依赖
pip install -r requirements.txt

# 运行打包脚本
python build_exe.py
```

## 打包配置说明

打包脚本会自动：

1. **安装 PyInstaller**：用于将 Python 项目打包成 exe
2. **创建 spec 文件**：配置打包参数
3. **包含必要文件**：
   - `config.yml` - 配置文件
   - `imgs/` - 图片资源目录
   - `session/` - 会话目录
   - `export/` - 导出目录
   - `logs/` - 日志目录
4. **包含隐藏依赖**：自动识别并包含所有必要的 Python 模块

## 打包输出

打包完成后，您将得到：

- `dist/CxKitty.exe` - 主程序文件
- `dist/` 目录下的其他必要文件

## 分发说明

分发时，您需要将整个 `dist` 目录打包分发给用户。用户只需要运行 `CxKitty.exe` 即可使用程序。

## 常见问题

### 1. 打包失败
- 确保 Python 版本为 3.11
- 确保所有依赖包安装成功
- 检查是否有足够的磁盘空间

### 2. exe 文件过大
- 这是正常现象，因为包含了 Python 运行环境和所有依赖
- 可以使用 UPX 压缩（已在配置中启用）

### 3. 运行时缺少文件
- 确保将整个 `dist` 目录一起分发
- 检查 `config.yml` 等配置文件是否被正确包含

## 注意事项

- 打包过程可能需要几分钟时间
- 生成的 exe 文件较大（约 100-200MB），这是正常现象
- 首次运行可能需要较长时间启动
- 建议在打包前先测试 Python 版本是否正常运行

## 技术支持

如果遇到打包问题，请检查：
1. Python 版本是否符合要求
2. 所有依赖是否正确安装
3. 项目文件是否完整
4. 系统权限是否足够
